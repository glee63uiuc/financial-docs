# Financial Document Manager - GitHub Pages Setup Guide

## Quick Setup (10 Minutes)

Follow these steps to host your Financial Document Manager on GitHub Pages and get a permanent web link.

## Step 1: Create a GitHub Account

1. Go to [https://github.com](https://github.com)
2. Click "Sign up" (top right)
3. Enter your email: `glee63uiuc@gmail.com` (or any email you prefer)
4. Create a password
5. Choose a username (e.g., `glee63` or `financial-docs-manager`)
6. Verify your email address

## Step 2: Create a New Repository

1. Once logged in, click the **"+"** icon (top right) â†’ **"New repository"**
2. Fill in the details:
   - **Repository name:** `financial-docs` (or any name you like)
   - **Description:** `Personal financial document manager`
   - **Public or Private:** Choose **Public** (required for free GitHub Pages)
   - âœ… Check **"Add a README file"**

---

## Step 3: Upload Your HTML File

1. In your new repository, click **"Add file"** â†’ **"Upload files"**
2. Drag and drop the `financial-doc-manager.html` file
   - OR click "choose your files" and select it
3. **IMPORTANT:** Rename the file to `index.html` (GitHub Pages requires this)
   - You can rename after uploading by clicking the pencil icon
4. Scroll down and click **"Commit changes"**

---

## Step 4: Enable GitHub Pages

1. In your repository, click **"Settings"** (top menu)
2. Scroll down the left sidebar and click **"Pages"** (under "Code and automation")
3. Under **"Source"**, select:
   - **Branch:** `main` (from dropdown)
   - **Folder:** `/ (root)`
4. Click **"Save"**
5. Wait 1-2 minutes for deployment
6. Refresh the page - you'll see a link like:
   ```
   Your site is live at https://yourusername.github.io/financial-docs/
   ```

## Step 5: Access Your App

1. **Copy the link** from Step 4
2. **Open it in Safari** on your iPhone/iPad/Mac
3. **Bookmark it** or **Add to Home Screen** (iPhone)

## Troubleshooting:

### "Page not found" error
- Wait 2-3 minutes after enabling GitHub Pages
- Make sure file is named `index.html` (not `financial-doc-manager.html`)
- Check that you selected `main` branch in Settings â†’ Pages

### "Repository is private" error
- GitHub Pages (free) requires public repositories
- Go to Settings â†’ General â†’ Danger Zone â†’ Change visibility â†’ Make public

### App won't load on iPhone
- Must open in Safari (not Chrome or other browsers)
- Clear Safari cache if needed
- Make sure you have internet connection (first load only)

### Lost the link?
- Go to your repository â†’ Settings â†’ Pages
- The link is displayed at the top
